下面我用**專案經理（PM）**視角，把「收件 → 編碼 → 賦予 RFID → 運送銀行端 → 評鑑狀態 → 入庫 → 上鍊」規劃成一套**可執行的端到端流程**（含角色分工、里程碑、資料欄位與控制點）。我也把您前面提到的 **ONS / Savant（中介層）**一起納入，讓編碼由系統主導、並可支援 DPP/上鍊的保管鏈（Chain of Custody）。

---

## 1) 端到端流程總覽（PM 的主流程 + 關卡）

### 流程泳道（Swimlane）角色

* **文物管理端（Museum/Owner）**：收件、初檢、編碼、貼標、出庫
* **物流端（Logistics）**：運送、環境監測、交接
* **銀行端（Bank Custody）**：接收、保管、評鑑協調
* **第三方評鑑（Appraiser/Lab）**：鑑定、出具報告
* **系統端（IT / Platform）**：編碼、ONS、Savant、CMS/WMS、區塊鏈

### 主流程（含關卡 Gate）

**G0 事前建檔（可選但建議）**
建立「預期入庫」清單（ASN）、授權文件、保險/運輸條款、評鑑規範與照片標準。

**Step 1 收到文物（入件）—G1：收件驗證**

1. 收件登錄：到件時間、來源、包裝外觀、封條狀態
2. 基本初檢：外觀與既有標記比對、拍照（含比例尺/色卡）
3. 建立「臨時件號（Temp Receipt ID）」並貼臨時條碼（避免先貼 RFID）
   ✅ G1 放行條件：文件齊、包裝完整、無明顯破損/異常；否則走「異常處理」

**Step 2 系統編碼（您要求：編碼由系統主導）—G2：編碼核准**

1. 平台生成 **Global Asset ID（內部主鍵）**
2. 生成 **EPC / UID 編碼**（建議採 EPC 架構：CompanyPrefix + ItemRef + Serial）
3. 在系統建立「資產主檔」：分類、材質、尺寸、權屬、保險值（若可）、保存條件等
4. ONS 設定：把 EPC 指向對應服務端點（例如：資產履歷服務、評鑑報告服務、DPP 服務）
   ✅ G2 放行條件：資產主檔必填欄位完整、權限核可、編碼未重複

**Step 3 賦予 RFID（貼標/封裝/關聯）—G3：貼標驗證**

1. 選型：UHF（物流/庫存）+（必要時）NFC（近距驗證）/ 主動式（高價即時定位）
2. 實體貼標策略：避開脆弱表面；採「中性載體」(tag-on-holder) 或「封條式防拆標」
3. 綁定關聯：掃描 RFID → 系統寫入/綁定 EPC ↔ Global Asset ID
4. 讀取測試：固定讀取器/手持器多角度測試；記錄 RSSI/可讀距離基線
   ✅ G3 放行條件：讀取成功率達標、標籤位置記錄完成、（若使用防拆）封條狀態 OK

**Step 4 出庫與運送至銀行端（交接 + 監測）—G4：出庫交接**

1. 出庫清點：RFID Tunnel 或手持批次核對「出庫清單」
2. 建立電子交接單（e-POD）：交接人、時間、封條、箱號、RFID 清單
3. 運輸監測：溫濕度/震動（若有感測標籤或箱內 logger）→ Savant 收集事件
   ✅ G4 放行條件：清單核對一致、封條完整、保險/運輸文件齊備

**Step 5 銀行端接收與保管（入庫前）—G5：銀行接收驗證**

1. 到件掃描：RFID Tunnel/手持掃描，核對預期 ASN
2. 封條檢查：若防拆標異常 → 自動觸發「高風險事件」
3. 臨時上架：進入「待評鑑/待確認」區（Quarantine / Hold）
   ✅ G5 放行條件：到件清單一致、封條/環境記錄無重大異常；否則進異常流程

**Step 6 評鑑狀態管理（第三方鑑定）—G6：評鑑結案**

1. 送評鑑派工：系統產生評鑑工單（Work Order）
2. 評鑑過程事件：開箱/檢測/封存各節點掃描 RFID 形成事件鏈
3. 上傳報告：鑑定報告（PDF/結構化欄位）、評級、影像、建議保存條件
4. 評鑑結果回寫：狀態改為「已評鑑」並生成「評鑑摘要雜湊（hash）」
   ✅ G6 放行條件：報告齊全、簽章/驗章完成、資料校驗通過

**Step 7 正式入庫（銀行端或指定庫房）—G7：入庫完成**

1. 指派庫位：庫房/層架/箱位（Location ID）
2. 入庫掃描：RFID 讀取確認 + 庫位綁定
3. 設定電子圍籬：限定可移動區域與授權流程（Geofencing / Zonal tracking）
   ✅ G7 放行條件：庫位綁定成功、盤點基線建立、權限策略生效

**Step 8 上鍊（DPP / NFT / Chain-of-Custody）—G8：上鍊成功**

1. 建立 DPP：把 EPC/Global Asset ID 與 DPP ID 關聯
2. 上鍊內容：不把敏感原文上鍊（避免洩漏），改上「指紋」：

   * 資產主檔摘要 hash
   * 評鑑報告 hash
   * 關鍵事件（收件/交接/評鑑結案/入庫）之事件 hash
3. 鏈上交易完成：產生 TxID，回寫平台；對外查驗以「掃描 RFID → 查 DPP」呈現
   ✅ G8 放行條件：Tx 確認完成、DPP 可查、權限與隱私策略符合要求

## 2) 狀態機（PM 必備：讓流程可控、可追蹤、可稽核）

建議資產狀態（Asset Status）：

1. `RECEIVED`（已收件）
2. `REGISTERED`（已建檔）
3. `ENCODED`（已編碼）
4. `TAGGED`（已貼標/綁定）
5. `DISPATCHED`（已出庫/運送中）
6. `BANK_RECEIVED`（銀行已接收）
7. `ON_HOLD`（待確認/隔離）
8. `IN_APPRAISAL`（評鑑中）
9. `APPRAISED`（已評鑑）
10. `STORED`（已入庫）
11. `ON_CHAIN`（已上鍊）
12. `EXCEPTION`（異常處理中）

每個狀態都必須由「RFID 掃描事件 + 權限人簽核」推進，避免人工口頭更新。

## 3) 系統落地：ONS / Savant / 事件資料流（您關心的編碼主導）

### 編碼主導（平台）

* 平台生成：`GlobalAssetID`、`EPC`、`DPP_ID`
* RFID 只承載 EPC/UID（可含少量必要 metadata），真正資料在後端服務

### ONS（解析與導流）

* EPC → ONS → 取得對應服務端點（例如：資產履歷 API、評鑑報告 API、DPP 查驗頁）
* 好處：未來服務遷移時不用重寫標籤，只更新 ONS

### Savant（RFID 中介層 / Edge Middleware）

* Reader 收到原始讀取 → Savant 進行：

  * 去重、濾波、事件聚合（如：進/出、停留、越界）
  * 產生標準事件：`ARRIVAL/DEPARTURE/OPEN_SEAL/TAMPER/ENV_ALERT`
* Savant 將事件送到：

  * CMS/WMS（庫存與庫位）
  * 風險告警系統（安防/保險）
  * 區塊鏈上鍊服務（只送 hash 與摘要）

## 4) 必填資料欄位（讓評鑑、入庫、上鍊不會卡）

### 資產主檔（Asset Master）

* GlobalAssetID、EPC、類別/材質、尺寸/重量、權屬、保存條件（T/RH/Lux）
* 影像（收件照、細部照）、包裝/箱號、保險與運輸條款（若可）

### 事件紀錄（Chain-of-Custody Event）

* EventID、時間、地點/區域、操作者、角色、掃描設備、封條狀態、清單核對結果
* 環境資料（若有）：T、RH、Shock、Tilt、Lux
* 對應文件 hash：交接單 hash、評鑑報告 hash

### 評鑑資料（Appraisal）

* 評鑑機構、評鑑人、方法、日期、結果等級、報告檔案、報告 hash、結論摘要

### 上鍊資料（On-chain Payload）

* DPP_ID、Asset Master hash、Appraisal hash、關鍵事件 hash、TxID

## 5) 里程碑與交付物（PM 管理用）

* **M1（收件流程上線）**：收件 SOP、初檢表、拍照規範、異常流程
* **M2（編碼/貼標完成）**：編碼規格、標籤選型、貼標規範、讀取驗收報告
* **M3（銀行交接閉環）**：ASN/e-POD、出入庫掃描、封條/防拆策略
* **M4（評鑑工單閉環）**：評鑑工單、報告模板、狀態回寫、hash 生成
* **M5（入庫+盤點）**：庫位模型、盤點策略、電子圍籬規則
* **M6（上鍊/DPP）**：上鍊策略（隱私）、智能合約/鏈上服務、Tx 回寫與查驗頁

---

## 6) 異常處理（必備，否則實務一定翻車）

常見異常與處置：

* **封條/防拆標異常**：立即 `ON_HOLD`，啟動雙人復核 + 影像紀錄 + 保險通知
* **RFID 讀不到**：切換備援讀取器/改掃 NFC/改人工條碼；必要時換標並建立「換標事件」
* **清單不一致（缺件/多件）**：凍結交接、建立差異單（Discrepancy Report）
* **環境超標**：自動告警→轉移到安全區→記錄處置事件→後續評鑑附註
